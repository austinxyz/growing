# 面试准备功能使用指南

## 📋 功能概述

面试准备分析功能可以根据职位JD和你的简历/项目/管理经验，智能生成针对性的面试准备计划，包括需要复习的技能、重点准备的项目和管理经验案例。

## 🎯 功能特点

1. **智能分析**: AI分析JD要求，匹配你的经验，生成准备重点
2. **结构化准备**: 按面试阶段组织准备内容
3. **优先级标注**: 高优先级项目自动标注，重点突出
4. **STARL框架**: 为项目和管理经验提供STARL叙述建议

## 🚀 使用步骤

1. 登录系统（用户: austinxu, 密码: helloworld）

   ```

   ```
4. 点击 **"复制"** 按钮，将prompt复制到剪贴板

### 步骤3: 在Claude Code中执行分析

1. 打开终端（Terminal）
2. 确保在项目根目录：`/Users/yanzxu/claude/growing`
3. 粘贴并执行刚才复制的prompt：
   ```bash
   Prepare for interview job 13 user 3
   ```
4. Claude Code会调用 `job-prepare` skill执行以下操作：
   - 查询JD详情（职位要求、职责）

   - AI分析匹配度，识别重点准备内容

### 步骤4: 查看准备计划

1. 执行完成后，刷新浏览器页面
2. 在 **"面试准备"** tab中查看完整准备计划

## 📊 准备计划结构

### 面试阶段 (Interview Stage)

每个面试阶段包含：

- **阶段名称**: 如"Overall Preparation"、"Phone Screen"、"Technical Round"
- **阶段顺序**: 面试流程中的顺序
- **技能数量**: 需要准备的技能领域数量
- **Focus Area数量**: 具体需要复习的知识点数量
- **准备笔记**: AI生成的总体准备建议

### 准备清单 (Checklist Items)

清单项分为以下类别：

#### 1. 项目 (Project)
- **内容**: 需要重点准备的项目案例
- **优先级**: 高/普通
- **准备要点**:
  - STARL框架叙述
  - 关键指标准备
  - 技术细节深入点
  - 可能的追问准备

**示例**:
```
✓ 项目: Cloud Native Migration (高优先级)

准备要点:
- Be ready to explain K8s operator and controller architecture in detail
- Prepare specific metrics: 2000+ apps migrated, 75% deployment time reduction
- Highlight cross-team coordination strategy
- Emphasize observability dashboard implementation

STARL框架:
S: eBay transitioning from VM to K8s, 2000+ apps need migration
T: Lead migration strategy, ensure zero downtime
A: Built K8s operators, established communication channels, created observability
R: 2000+ apps migrated, 75% faster deployments, improved reliability
L: Cross-team communication and incremental approach are critical
```

#### 2. 管理经验 (Management)
- **内容**: 需要准备的管理案例（STARL格式）
- **优先级**: 高/普通
- **准备要点**:
  - 完整STARL叙述
  - 关键指标
  - 匹配的JD要求

**示例**:
```
✓ 管理: Yiran's Growing - K8s Upgrade & Staff Promotion (高优先级)

准备要点:
- Prepare full STARL narrative with promotion as key result
- Emphasize coaching strategy: training, architecture review, feedback
- Quantify impact: K8s upgrade delivered, engineer promoted

关键指标:
- Starting point: Senior engineer, 80% ready for staff
- Project outcome: K8s upgrade successfully delivered
- Promotion outcome: Engineer promoted to staff level
```

#### 3. 技能准备 (Skill)
- **技能名称**: 如"System Design"、"People Management"
- **Focus Areas**: 需要复习的具体知识点
- **准备理由**: 为什么这个技能对此职位重要

**示例**:
```
技能: System Design - Observability & Operational Excellence

理由: JD emphasizes "reliable and secure execution of workloads"

Focus Areas:
- Observability (10, 11): Dashboard implementation, monitoring strategy
- Operational Excellence (13): Host-level resource management
- Automation & Tooling (28): Platform building
```

#### 4. 技术深入 (Technical)
- **内容**: 需要深入复习的技术领域
- **准备重点**: 具体技术知识点

**示例**:
```
技术深入: Kubernetes Fundamentals (高优先级)

准备:
- K8s architecture: operators, controllers, CRDs, admission webhooks
- Container technologies: Docker, containerd, runtime security
- Host-level resource management: cgroups, namespaces, isolation
- Security: Pod security policies, RBAC, network policies
```

#### 5. 其他 (Other)
- **内容**: 初筛、文化契合等其他准备要点
- **示例**: Recruiter talking points, Why this company, Career overview

## 💡 使用技巧

### 1. 面试阶段识别

如果你已经定义了面试阶段（如"Phone Screen"、"Technical"、"Behavioral"），job-prepare skill会智能分配准备内容：

- **Phone Screen**: 管理经验 + 通用技能
- **Technical Interview**: 技术项目 + 技术Focus Areas
- **System Design Round**: 系统设计项目 + 分布式系统
- **Behavioral Round**: 所有管理经验 + 领导力Focus Areas
- **Onsite**: 剩余所有内容

### 2. 准备优先级

- **🟡 高优先级**: 最相关的项目和经验，优先准备
- **⚪ 普通**: 辅助性内容，有时间再准备

### 3. 重复执行

可以多次执行面试准备分析：
- 第一次执行：创建默认"Overall Preparation"阶段
- 更新JD或经验后：重新执行，获得更新的建议

### 4. 补充数据

如果发现准备计划中提示"缺少XX案例"，可以：
1. 在"项目经验"或"人员管理经验"页面添加相关案例
2. 重新执行面试准备分析
3. 查看更新后的准备计划

## 🔧 技术细节

### 数据库表
- `interview_stages`: 存储面试阶段信息
- `interview_preparation_checklist`: 存储准备清单项

### API Endpoints
- `GET /api/interview-stages/job-application/{jobApplicationId}`: 获取面试阶段
- `GET /api/interview-preparation-checklist/stage/{stageId}`: 获取准备清单

### Skill配置
- **Skill名称**: job-prepare
- **位置**: `.claude/skills/job-prepare/`
- **输入参数**: job_id, user_id
- **输出**: JSON格式的准备计划

## ❓ 常见问题

### Q1: 为什么没有显示准备计划？
**A**: 确保以下步骤都完成：
1. 生成并复制了prompt
2. 在Claude Code中执行了prompt
3. 执行成功（没有报错）
4. 刷新了浏览器页面

### Q2: 可以修改准备计划吗？
**A**: 目前准备计划是AI自动生成的。如果需要修改：
- 可以在"面试流程"tab中手动创建/编辑面试阶段
- 未来版本会支持直接编辑准备清单

### Q3: 支持哪些类型的面试阶段？
**A**:
- 自动识别：Phone Screen, Technical, System Design, Behavioral, Onsite
- 也可以创建自定义阶段名称

### Q4: 准备计划会覆盖已有数据吗？
**A**:
- 第一次执行：创建新阶段
- 已有阶段：更新现有阶段，添加新的清单项
- 不会删除手动创建的内容
