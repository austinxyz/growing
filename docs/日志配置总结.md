## 🚀 使用方式

### 开发环境
```bash
cd backend
./start.sh          # 默认dev模式
./start.sh dev      # 显式指定
```

**特征**：
- 控制台显示SQL语句
- 可以看到SQL参数绑定
- Swagger UI可访问：http://localhost:8082/api/swagger-ui.html
- 详细错误堆栈

### 生产环境
```bash
cd backend
./start.sh prod     # 生产模式
```

**特征**：
- 控制台无SQL语句
- 只记录WARN和ERROR
- Swagger UI禁用（404）
- 错误信息精简

## 🔧 日志等级详解

从低到高（输出量从多到少）：

1. **TRACE** - 最详细（开发环境：SQL参数绑定）
2. **DEBUG** - 调试信息（开发环境：Hibernate统计，应用DEBUG）
3. **INFO** - 常规信息（生产环境：应用日志）
4. **WARN** - 警告（生产环境：Hibernate/Spring）
5. **ERROR** - 错误（所有环境）

## ⚠️ 注意事项

1. **NEVER在生产环境启用dev配置** - 会导致性能下降10-20%
2. **调试SQL时临时启用** - 调试完毕后务必改回去
3. **日志文件轮转** - 生产环境建议启用文件日志并配置轮转
4. **敏感信息保护** - SQL参数可能包含用户数据，生产环境必须关闭
