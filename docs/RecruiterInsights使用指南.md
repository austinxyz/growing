# Recruiter Insights 使用指南

## 功能概述

Recruiter Insights 功能允许你记录从招聘人员（Recruiter）那里获得的关于职位、团队和面试流程的内部信息。这些信息对准备面试非常有价值。

## 位置

**求职-面试-申请列表** → 选择职位 → **概览** Tab → **Recruiter Insights** 区域

## 可记录的信息

### 1. 团队规模 (Team Size)
- **示例**: "10-15人", "20+人", "5人小团队"
- **用途**: 了解团队大小，判断合作方式（小团队更灵活，大团队分工更明确）

### 2. 团队文化 (Team Culture)
- **示例**: "扁平化管理，鼓励创新，工作氛围轻松"
- **用途**: 判断是否符合自己的工作风格

### 3. 技术栈偏好 (Tech Stack Preference)
- **输入方式**: 输入技术栈后按 Enter 或逗号添加
- **示例**: Java, Spring Boot, Redis, Kubernetes, AWS
- **用途**: 提前准备相关技术问题，强调匹配的技术经验

### 4. 面试重点 (Interview Focus)
- **示例**: "系统设计和编程基础，特别关注分布式系统经验"
- **用途**: 针对性准备，提高面试通过率

### 5. 流程Tips (Process Tips)
- **示例**: "第一轮技术面试会很深入，建议准备常见算法题。系统设计轮会问实际项目经验，需要准备至少2个完整案例。"
- **用途**: 了解面试流程和评估标准

## 使用步骤

### 添加 Recruiter Insights

1. 进入"求职-面试-申请列表"页面
2. 选择一个职位
3. 在"概览"Tab中找到"Recruiter Insights"区域
4. 点击"添加"按钮
5. 填写Recruiter提供的信息
6. 点击"保存"

### 编辑 Recruiter Insights

1. 在已有的"Recruiter Insights"区域
2. 点击"编辑"按钮
3. 修改信息
4. 点击"保存"

### 技术栈输入技巧

在"技术栈偏好"字段：
- 输入技术名称后按 **Enter** 添加
- 或者输入逗号后自动添加
- 点击标签上的 **×** 删除

**示例输入流程**：
1. 输入 "Java" → 按Enter
2. 输入 "Spring Boot" → 按Enter
3. 输入 "Redis, Kubernetes" → 按Enter （可以批量添加）

## 实际使用场景

### 场景1：与Recruiter初次沟通后

Recruiter说："我们团队是一个10人的全栈团队，主要使用Java和React。团队氛围很友好，鼓励技术创新。面试主要看系统设计能力和编程基础。"

**记录到系统**：
- **团队规模**: 10人
- **团队文化**: 全栈团队，友好氛围，鼓励技术创新
- **技术栈偏好**: Java, React, Spring Boot
- **面试重点**: 系统设计能力和编程基础

### 场景2：内推前了解信息

内推人说："这个组做云基础设施，主要用Kubernetes和Go。面试会问很多分布式系统的问题，建议准备CAP定理、一致性协议这些。第一轮是算法，难度中等偏上。系统设计轮会深入聊你做过的项目。"

**记录到系统**：
- **团队规模**: （内推人未提供）
- **团队文化**: 云基础设施团队
- **技术栈偏好**: Kubernetes, Go, 分布式系统
- **面试重点**: 分布式系统（CAP、一致性协议）
- **流程Tips**: 第一轮算法题难度中等偏上。系统设计轮会深入聊项目，需要准备完整案例。建议重点复习CAP定理和一致性协议。

### 场景3：面试前Recruiter电话

Recruiter电话说："明天的面试官是我们的Tech Lead，他比较看重实际项目经验。如果你有分布式系统的经验，记得在简历里highlight一下。还有，他不太喜欢纯理论的回答，最好能结合你做过的项目说。"

**记录到系统**：
- **面试重点**: 实际项目经验，分布式系统
- **流程Tips**: 面试官是Tech Lead，看重实践经验。避免纯理论回答，结合项目经验说明。建议提前准备2-3个分布式系统相关的项目案例。

## 为什么要记录 Recruiter Insights？

### 1. **内部信息优势**
Recruiter 提供的信息是外部获取不到的，比如：
- 团队真实的技术栈（不是JD上笼统的描述）
- 面试官的偏好和评估标准
- 面试难度和侧重点

### 2. **针对性准备**
知道面试重点后可以：
- 集中精力准备相关内容
- 避免浪费时间在不重要的方面
- 提高面试通过率

### 3. **简历定制**
根据技术栈偏好调整简历：
- Highlight 匹配的技术经验
- 准备相关项目案例

### 4. **评估Offer**
了解团队文化和规模后：
- 判断是否符合职业发展规划
- 对比不同公司的团队氛围

## Tips

1. **及时记录**: 与Recruiter沟通后立即记录，避免遗忘
2. **详细记录**: 越详细越好，包括具体的技术点、例子
3. **标注来源**: 如果是内推人提供的，可以在备注里标注
4. **持续更新**: 每次沟通后更新，积累更多信息
5. **对比信息**: 不同来源的信息可能有出入，记录下来方便对比

## 数据隐私

- Recruiter Insights 数据存储在 `job_applications` 表的 `recruiter_insights` JSON 字段
- 只有你自己可以查看和编辑
- 不会与其他用户共享
