# Phase 2 - 技能体系管理详细需求

> **版本**: v1.0
> **日期**: 2025-12-20
> **状态**: ✅ 已实施完成

## 一、需求概述

### 1.1 目标

为Growing个人成长管理系统构建完整的技能体系管理功能，支持管理员配置技能树和学习资源，用户查看个人职业路径下的技能并添加个人学习资源。

### 1.2 核心概念

```
职业路径 (Career Path) - Phase 1已实现
  └─ 技能 (Skill) - Phase 2新增
       ├─ 专注领域 (Focus Area) - Phase 2新增
       └─ 学习资源 (Learning Resource) - Phase 2新增
```

**关键设计决策**：
- **Focus Area**：纯展示性质，只包含名称和描述，不关联其他数据
- **Learning Resource**：直接归属于技能，不归属于Focus Area
- **资源可见性**：用户添加的资源只有自己可见，不影响其他用户

### 1.3 用户角色

| 角色 | 权限 |
|------|------|
| **管理员** | 创建/编辑/删除技能、Focus Area、官方资源；关联技能到职业路径 |
| **普通用户** | 查看技能详情；添加/删除个人学习资源 |

## 二、功能需求

### 2.1 用户端功能

#### 2.1.1 查看技能列表

**功能描述**：用户登录后可以查看自己职业路径下的所有技能

**页面路径**：`/skills/career-paths`

**展示内容**：
- 职业路径Tab切换（用户可能有多个职业路径）
- 技能卡片网格布局，每个卡片显示：
  - 技能图标emoji
  - 技能名称
  - 技能描述
  - 重要标记（⭐）
  - 专注领域数量
  - 学习资源数量

**交互行为**：
- 点击技能卡片跳转到技能详情页
- 支持"全部"和"重要技能"过滤

#### 2.1.2 查看技能详情

**功能描述**：查看某个技能的详细信息

**页面路径**：`/skills/{id}`

**Tab 1: 专注领域**
- 以列表形式展示该技能下的所有专注领域
- 每个领域包含：
  - 序号
  - 领域名称
  - 领域描述（可展开/收起）

**Tab 2: 学习资源**
- 按资源类型分组显示（书籍、课程、网站、博客等）
- 每个资源显示：
  - 资源类型徽章
  - 资源标题
  - 作者/来源
  - 描述
  - 链接（可点击）
  - 官方/个人标记
- 用户添加的资源显示"删除"按钮
- 顶部有"+ 添加我的资源"按钮

#### 2.1.3 添加个人学习资源

**功能描述**：用户为技能添加自己发现的学习资源

**触发方式**：点击"+ 添加我的资源"按钮

**弹窗表单**：
- 资源标题（必填）
- 资源类型（下拉选择：文档/视频/课程/书籍/博客/练习/工具/文章/网站/其他）
- 资源链接（选填）
- 作者/来源（选填）
- 描述（选填）

**权限规则**：
- 用户只能看到官方资源和自己添加的资源
- 用户添加的资源标记为`isOfficial=false`
- 用户只能删除自己添加的资源

### 2.2 管理员功能

#### 2.2.1 技能管理列表

**功能描述**：管理所有技能

**页面路径**：`/admin/skills`（或通过Settings → 技能管理进入）

**页面布局**：
- **左侧面板** (384px宽)：
  - 职业路径Tab（垂直排列）
  - 每个Tab显示职业路径名称、图标和技能数量
  - 点击Tab后显示该职业路径下的技能列表
  - 技能列表显示：技能图标、名称、重要标记
  - 底部"创建新技能"按钮

- **右侧面板** (占据剩余空间)：
  - 未选中技能时：提示"选择一个技能查看详情"
  - 选中技能后：显示技能详细信息（3个Card）

#### 2.2.2 技能详情管理

**Card 1: 基本信息**
- 技能图标（emoji选择器）
- 技能名称
- 描述
- 是否重要
- 关联的职业路径（可能多个）
- 统计信息（专注领域数、资源数、显示顺序）
- 编辑/删除按钮

**Card 2: 专注领域**
- 显示数量
- 添加领域按钮
- 领域列表，每项显示：
  - 序号
  - 领域名称
  - 描述
  - 编辑/删除按钮

**Card 3: 学习资源**
- 显示数量
- 添加资源按钮
- 资源列表，每项显示：
  - 资源类型徽章
  - 官方标记（如果是官方资源）
  - 标题
  - 描述
  - 作者
  - 链接
  - 编辑/删除按钮

#### 2.2.3 创建/编辑技能

**弹窗表单**：
- 技能名称（必填）
- 描述
- 是否重要（复选框）
- 图标Emoji（emoji选择器，50个常用emoji + 手动输入）
- 关联职业路径（多选复选框）

**保存逻辑**：
- 创建时自动关联到选中的职业路径
- 编辑时可以修改职业路径关联

#### 2.2.4 管理专注领域

**添加/编辑弹窗**：
- 领域名称（必填）
- 描述

**删除确认**：
- 弹出确认对话框："确定要删除这个专注领域吗？"

#### 2.2.5 管理学习资源

**添加/编辑弹窗**：
- 资源标题（必填）
- 资源类型（下拉选择，10种类型）
- 资源链接
- 作者/来源
- 描述
- 是否官方（默认勾选，管理员添加的默认为官方）

**删除确认**：
- 弹出确认对话框："确定要删除这个学习资源吗？"

## 三、非功能需求

### 3.1 性能要求

- 技能列表页加载时间 < 1秒
- 技能详情页加载时间 < 500ms
- 支持100+技能、1000+专注领域、5000+学习资源

### 3.2 数据完整性

- 删除技能时级联删除关联的Focus Area和Learning Resource
- 删除用户时级联删除其创建的Learning Resource
- 职业路径与技能的多对多关系通过`career_path_skills`表维护

### 3.3 用户体验

- 所有删除操作需要二次确认
- 所有表单提交有Loading状态和成功/失败提示
- 支持响应式布局（桌面/平板/手机）
- 使用弹窗进行创建/编辑操作，避免页面跳转

### 3.4 安全性

- 用户只能删除自己添加的资源
- 管理员API需要admin角色验证
- 资源可见性按用户ID隔离

## 四、数据约束

### 4.1 字段约束

| 表名 | 字段 | 约束 |
|------|------|------|
| skills | name | NOT NULL, 最大100字符 |
| skills | icon | 最大50字符（emoji） |
| focus_areas | name | NOT NULL, 最大200字符 |
| learning_resources | title | NOT NULL, 最大200字符 |
| learning_resources | url | 最大500字符 |
| learning_resources | resource_type | ENUM枚举值 |

### 4.2 唯一性约束

| 表名 | 约束 | 说明 |
|------|------|------|
| career_path_skills | (career_path_id, skill_id) | 职业路径和技能组合唯一 |

### 4.3 级联删除

```
Career Path
  └─ career_path_skills (CASCADE)
       └─ Skill

Skill
  ├─ career_path_skills (CASCADE)
  ├─ focus_areas (CASCADE)
  └─ learning_resources (CASCADE)

User
  └─ learning_resources.created_by_user_id (CASCADE)
```

## 五、业务规则

### 5.1 资源可见性规则

```sql
-- 用户查看技能时，只能看到：
SELECT * FROM learning_resources
WHERE skill_id = ?
  AND (is_official = true OR created_by_user_id = ?)
ORDER BY display_order ASC, created_at ASC
```

### 5.2 资源类型枚举值

```
BOOK        - 书籍
COURSE      - 课程
WEBSITE     - 网站
ARTICLE     - 文章
VIDEO       - 视频
DOCUMENT    - 文档
BLOG        - 博客
PRACTICE    - 练习
TOOL        - 工具
OTHER       - 其他
```

### 5.3 显示顺序规则

- 技能：按`display_order`升序，相同则按ID升序
- 专注领域：按`display_order`升序，相同则按ID升序
- 学习资源：官方资源优先，然后按`display_order`和创建时间排序

### 5.4 重要技能标记

- `is_important = true`的技能会在列表页显示⭐标记
- 用户可以通过"重要技能"过滤器只查看重要技能

## 八、已知限制

1. **Focus Area暂不支持难度级别**：未来版本可能添加
2. **题库未关联到Focus Area**：题库功能将在Phase 3实现
3. **资源审核机制**：用户添加的资源不需要审核，直接可见（仅自己可见）
4. **资源排序**：暂不支持拖拽排序，使用`display_order`字段预留
5. **技能树可视化**：暂不支持，列表展示即可
