# Phase 4 详细需求 - 编程与数据结构学习模块

> **需求来源**: 用户需求 (2025-12-21)
> **目标用户**: 准备算法面试的工程师
> **核心价值**: 提供系统化的编程与数据结构学习路径，整合LeetCode、labuladong等资源
> **状态**: 需求撰写中 (待Review)

## 1. 背景与目标

### 1.1 用户痛点

**现状**:
- 用户学习编程与数据结构时，资源分散在多个平台：
  - LeetCode - 刷题平台
  - labuladong的算法笔记 - 系统化学习
  - Hello Interview - 题解参考
  - 极客时间王铮的算法课 - 视频课程
- 缺乏统一的学习入口和进度跟踪
- 难以建立知识体系，容易遗忘关键思路

**目标**:
- 在Growing App中整合所有学习资源
- 按照labuladong的结构组织学习内容
- 提供题目总结和笔记功能
- 维护常用算法模板库

### 1.2 成功标准

**功能完整性**:
- ✅ 支持按labuladong的6大分类浏览学习内容
- ✅ 管理员可以维护学习资料和题目关联
- ✅ 用户可以查看题目总结（LeetCode链接、关键思路等）
- ✅ 用户可以添加个人笔记
- ✅ 提供常用算法模板查询功能

**用户体验**:
- ✅ 清晰的知识结构（大分类 → 小分类 → 学习资料 → 题目）
- ✅ 一键跳转到LeetCode、labuladong等外部资源
- ✅ 支持Markdown格式的关键思路显示
- ✅ 学习进度可视化

## 2. 核心概念

### 2.1 labuladong的内容结构

**层级关系**:
```
大分类（6个固定分类）
  ├─ 基础篇
  ├─ 核心刷题框架
  ├─ 经典数据结构算法
  ├─ 经典暴力搜索算法
  ├─ 经典动态规划算法
  └─ 其他算法
      ↓
小分类（若干，如"数组"、"链表"、"二叉树"）
      ↓
学习资料（一篇或多篇文章/视频）
      ↓
题目集（多个LeetCode题目）
```

**示例**:
- 大分类: "核心刷题框架"
- 小分类: "双指针"
- 学习资料:
  - labuladong文章《双指针技巧总结》
  - 极客时间视频《双指针算法精讲》
- 题目集:
  - [26] 删除有序数组中的重复项
  - [27] 移除元素
  - [283] 移动零
  - ...

### 2.2 与现有结构的映射

**复用现有概念**:
- **技能 (Skill)** = "编程与数据结构"
- **Focus Area** = 小分类（如"数组"、"链表"）
- **Question** = LeetCode题目

**新增概念**:
- **大分类** - labuladong的6个主分类
- **学习资料** - 文章、视频等外部资源（需要新建表关联到Focus Area，区别于现有技能级别的学习资源）
- **题目扩展信息** - LeetCode链接、labuladong链接、关键思路等
- **算法模板** - 常用算法的代码模板和注意事项

#### 故事4: 维护常用算法模板

**角色**: 管理员
**目标**: 创建和管理常用算法模板库

**场景**:
1. 进入"设置 - 内容管理 - 常用算法"
2. 左侧显示算法模板列表（按分类组织，如"查找"、"排序"）
3. 点击"新建模板"按钮
4. 填写：
   - 名称: 二分查找
   - 分类: 查找
   - 简介: 在有序数组中查找目标值的经典算法
   - 模板代码: (Markdown编辑器，支持代码高亮)
     ```java
     public int binarySearch(int[] nums, int target) {
         int left = 0, right = nums.length - 1;
         while (left <= right) {
             int mid = left + (right - left) / 2;
             if (nums[mid] == target) return mid;
             else if (nums[mid] < target) left = mid + 1;
             else right = mid - 1;
         }
         return -1;
     }
     ```
   - 注意事项: (Markdown编辑器)
     ```
     - 数组必须有序
     - 注意边界条件 (left <= right)
     - 防止溢出: mid = left + (right - left) / 2
     ```
   - 时间复杂度: O(log n)
   - 空间复杂度: O(1)
   - 相关题目: 搜索选择 [704] 二分查找, [35] 搜索插入位置
5. 保存后，模板出现在列表中

**验收标准**:
- 支持Markdown格式的代码和注意事项
- 代码支持语法高亮
- 相关题目支持搜索选择
- 左侧列表支持按分类筛选

## 4. 功能需求

## 5. 非功能需求

### 5.1 性能要求

- 题目列表分页加载（每页50题）
- 大分类列表缓存（很少变化）
- Markdown渲染性能优化（大文档分块渲染）

### 5.2 安全要求

- 管理端所有API需要管理员权限
- 用户只能管理自己的笔记
- 防止XSS攻击（Markdown内容过滤）

### 5.3 兼容性要求

- 不影响现有的试题库功能
- Focus Area表的扩展字段对现有数据兼容
- 支持主流浏览器（Chrome、Safari、Firefox）

### 5.4 可维护性要求

- 大分类固定为6个，写死在代码中（Migration初始化）
- 小分类、学习资料、题目都支持CRUD
- 扩展信息采用UPSERT逻辑（不存在则创建，存在则更新）

## 7. UI/UX需求

### 7.1 视觉设计

- 延续现有风格（Tailwind CSS）
- 大分类Tab使用蓝色高亮
- 完成的题目显示绿色勾选图标
- 难度标签使用颜色区分（EASY绿色、MEDIUM黄色、HARD红色）

### 7.2 交互设计

- 题目默认折叠，点击标题或"展开"按钮展开
- 拖拽排序提供视觉反馈（高亮拖拽项）
- 外部链接点击在新窗口打开
- Markdown编辑器提供实时预览

### 7.3 响应式设计

- 支持桌面端（最小宽度1280px）
- 三栏布局在小屏幕上自适应为上下布局

## 9. 待澄清问题

以下问题需要在设计阶段进一步讨论：

1. **大分类的存储方式** ✅ 已明确
   - 需要独立的大分类表（major_categories）
   - 需要关联表（major_category_focus_areas）记录大分类和focus area的关系
   - 不修改现有的focus_areas表结构

2. **题目完成状态的判断** ✅ 已明确
   - Phase 4 不实现完成状态功能
   - 学习进度功能留待未来Phase实现
   - 不添加completed字段

3. **学习资料与题目的关联** ✅ 已明确
   - 学习资料通过新表（focus_area_learning_resources）关联到Focus Area
   - 题目本身包含其他链接字段（LeetCode、labuladong等）
   - 不需要学习资料和题目的显式关联

4. **标签和类似题目的存储** ✅ 已明确
   - 使用JSON数组存储，不创建独立的tags表
   - 类似题目也使用JSON数组存储题目ID
   - 简化数据模型，提高灵活性

5. **算法模板的分类** ✅ 已明确
   - 算法模板不需要分类功能
   - 左侧直接显示算法列表
   - algorithm_templates表不包含category字段
