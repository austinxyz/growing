# Phase 7 需求概要 - 求职管理模块

> **版本**: v1.0
> **创建时间**: 2026-01-03
> **状态**: 📋 需求阶段

---

## 1. 模块定位

**核心价值**: 将 Phase 1-6 的学习成果（技能、项目经验、试题答案）转化为求职竞争力，提供从简历准备到面试跟踪的全流程管理。

**与现有模块关系**:
- **Phase 2-6 输出**: 技能树、学习笔记、试题答案 → **Phase 7 输入**: 简历素材、面试准备
- **闭环**: 学习 → 实战 → 求职 → 反馈 → 改进学习方向

---

## 2. 核心功能模块

### 2.1 简历管理 (Resume Management)

**目标**: 结构化存储简历信息，支持多版本管理和定制化输出

**核心内容**:
1. **About** - 个人简介/职业目标
2. **Experience** - 工作经历
   - 公司、职位、时期
   - 职责描述
   - 重要项目（关联项目经验库）
   - 主要成就（量化数据）
3. **Contact & Links** - 联系方式、LinkedIn、GitHub、个人网站
4. **Skills** - 技能清单
   - 技术技能（编程语言、框架、数据库、云平台等）
   - 软技能（领导力、沟通、项目管理等）
   - 工具技能（Git、Docker、CI/CD 等）
   - 可选择性地从 Phase 2-6 技能树导入部分内容作为参考
5. **Education & Training** - 学历、培训、证书
6. **Languages** - 语言能力
7. **Hobbies** - 个人爱好

**关键特性**:
- 支持多版本简历（针对不同公司/职位定制）
- 可从公司职位 JD 自动生成定制简历
- 导出格式：PDF、Word、纯文本

---

### 2.2 项目经验库 (Project Experience Bank)

**目标**: 系统化总结项目经验，支持 Behavioral/Project Management 问题准备

**结构化框架**:

#### 技术项目模板
1. **What/When/Who/Why** - 项目背景
2. **Problem Statement & Challenges** - 问题定义和挑战
3. **How** - 解决方案
   - 技术选型和理由
   - 创新点/差异化做法
   - 个人贡献
4. **Result** - 成果
   - 量化数据（性能提升、成本节约、用户增长等）
   - 业务影响
   - 经验教训 (Lessons Learned)

#### 人员管理经验模板
1. **Manage Up** - 向上管理案例
2. **Cross-team Management** - 跨团队协作案例
3. **Team Growth** - 团队建设
   - Hiring（招聘经验）
   - High Performer（培养高绩效员工）
   - Low Performer（处理低绩效员工）

**关联功能**:
- 项目经验可关联到简历的 Experience 部分
- 可作为 Behavioral 问题的答案素材（关联 Phase 6 试题库）

---

### 2.3 公司求职档案 (Company Job Tracking)

**目标**: 管理目标公司信息和求职全流程，提高面试准备效率

**核心信息**:

#### 2.3.1 公司基本信息
1. **公司简介** - 业务领域、规模、发展历史
2. **企业文化** - 价值观、工作氛围（关联 Behavioral 问题准备）
3. **有用链接** - 技术博客、工程博客、公司官网、Glassdoor 等

#### 2.3.2 职位管理（一个公司多个职位）
1. **JD (Job Description)**
   - 职位名称、发布时间、职位链接
   - **Qualifications**（最重要）- 技能要求
   - **Responsibilities**（最重要）- 岗位职责
2. **面试流程** - 每个阶段的准备重点
   - Phone Screen → 关联技能（如 Phase 4 算法、Phase 6 Behavioral）
   - Onsite Rounds → 每轮主题（系统设计、技术深度、领导力等）
3. **定制简历** - 根据 JD 自动生成/手动调整
4. **Recruiter** - 招聘人员联系方式、沟通记录

#### 2.3.3 面试跟踪
1. **准备阶段** - 关联面试流程和技能准备清单
2. **面试记录** - 每轮面试
   - 面试官信息
   - 问题列表（可关联 Phase 3-6 试题库）
   - 自我评估
   - 面试官反馈
3. **结果跟踪** - Offer/Reject/Pending，拒绝原因分析

#### 2.3.4 人脉管理
- **内部推荐人** - 前同事、同学、朋友
- 推荐状态跟踪

#### 2.3.5 面经参考
- 其他人的面试经验（外部资源链接或笔记）

## 3. 数据关联设计

### 3.1 与现有模块的关联

| Phase 7 模块 | 关联模块 | 关联方式 |
|-------------|---------|---------|
| 简历 Skills | Phase 2-6 技能树 | 可选择性导入作为参考（简历技能更侧重具体技术栈） |
| 项目经验 | Phase 6 Behavioral 试题 | 作为 STAR 答案素材 |
| 公司面试问题 | Phase 3-6 试题库 | 标记"此题在 XX 公司被问到" |
| 定制简历 | JD Qualifications | 自动匹配技能和经验 |
| 面试准备清单 | Phase 4/5/6 学习内容 | 生成复习计划 |

### 3.2 智能推荐功能（可选）

- **JD 技能匹配度分析** - 对比简历技能和 JD 要求，给出匹配度评分
- **简历优化建议** - 根据 JD 关键词推荐调整简历重点
- **高频面试题推荐** - 基于公司和职位类型推荐试题

## 4. 用户交互流程

### 4.1 求职准备流程
```
1. 创建/更新基础简历 (Resume Management)
2. 总结项目经验 (Project Experience Bank)
3. 添加目标公司 (Company Profile)
4. 创建职位申请 (Job Application)
5. 分析 JD → 生成定制简历
6. 查看面试流程 → 制定准备计划（关联 Phase 2-6 内容）
7. 记录面试过程 → 更新面试记录
8. 收到反馈 → 分析改进方向
```

### 4.2 面试复盘流程
```
1. 查看面试记录中被问到的问题
2. 检查答案质量（是否用到项目经验？STAR 是否完整？）
3. 如果失败：分析薄弱技能 → 回到 Phase 2-6 加强学习
4. 更新项目经验库（补充新的案例细节）
```

## 5. Phase 7.1 实施范围与页面设计

### 5.1 侧边栏导航结构

```
求职
├─ 简历
│  ├─ 基本信息管理
│  ├─ 项目经验库
│  └─ 人员管理经验库
└─ 面试
   ├─ 公司与职位
   ├─ 定制简历
   └─ 面试记录
```

### 5.2 页面功能明细

#### 求职 → 简历

**5.2.1 基本信息管理页面**
- 功能：管理简历基本信息（About, Contact, Skills, Education, Languages, Hobbies）
- 支持：多版本简历管理
- 导出：PDF/Word/纯文本

**5.2.2 项目经验库页面**
- 功能：管理技术项目经验（What/When/Who/Why, Problem, How, Result）
- 关联：可关联到简历 Experience 部分
- 用途：Behavioral 问题答案素材

**5.2.3 人员管理经验库页面**
- 功能：管理人员管理经验（Manage Up, Cross-team, Team Growth）
- 关联：可关联到简历 Experience 部分
- 用途：People Management 问题答案素材

#### 求职 → 面试

**5.2.4 公司与职位页面**
- 公司信息：简介、企业文化、有用链接
- 职位管理：JD（Qualifications + Responsibilities）、面试流程
- Recruiter 管理：招聘人员联系方式、沟通记录
- 人脉管理：内推人脉、推荐状态跟踪

**5.2.5 定制简历页面**
- 功能：根据 JD 生成定制简历
- 自动匹配：JD Qualifications ↔ 简历 Skills + 项目经验
- 编辑：手动调整简历内容
- 导出：PDF/Word 格式

**5.2.6 面试记录跟踪页面**
- 准备阶段：关联面试流程和技能准备清单
- 面试记录：面试官信息、问题列表（关联 Phase 3-6 试题库）、自我评估、反馈
- 结果跟踪：Offer/Reject/Pending，拒绝原因分析

- ✅ 简历基本信息管理

- ✅ 公司信息 + 职位管理
- ✅ Recruiter 管理
- ✅ 人脉推荐管理
- ✅ 定制简历生成
- ✅ 面试记录跟踪

#### P1 (未来扩展)

- 简历优化建议（AI 推荐）
- 高频面试题推荐（基于公司和职位）
- 面试准备清单自动生成

## 6. 技术考虑

### 6.1 数据库设计
- `resumes` - 简历主表（支持多版本）
- `resume_experiences` - 工作经历
- `project_experiences` - 项目经验库
- `management_experiences` - 人员管理经验库
- `companies` - 公司档案
- `job_applications` - 职位申请
- `interview_records` - 面试记录
- `referrals` - 内推人脉

### 6.2 数据导入/导出
- **导入**: 从 LinkedIn 导入基本信息（可选）
- **导出**: PDF/Word/纯文本简历
- **参考**: 可查看 Phase 2-6 技能树作为简历技能填写参考

### 6.3 隐私安全
- 简历数据加密存储
- 敏感信息（联系方式、公司内部信息）访问控制
- 支持简历匿名化（隐藏个人联系方式用于分享）

## 7. 成功指标

| 指标 | 目标 |
|-----|------|
| 简历版本管理 | 支持至少 10 个定制版本 |
| 项目经验覆盖率 | 至少 80% 工作经历关联项目经验 |
| 面试准备效率 | JD 分析 → 定制简历 ≤ 30 分钟 |
| 面试复盘率 | 每次面试后 100% 记录问题和反馈 |
| 技能匹配准确度 | JD 匹配分析准确率 ≥ 85% |
